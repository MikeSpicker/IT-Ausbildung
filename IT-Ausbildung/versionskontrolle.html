Grundlagen der Versionskontrolle

Was ist Versionskontrolle?

Versionskontrolle ist ein System, das Änderungen an Dateien über die Zeit verfolgt. Es ermöglicht:

  • Nachverfolgung von Änderungen
  • Wiederherstellung älterer Versionen
  • Parallele Entwicklung durch mehrere Personen
  • Dokumentation von Entscheidungen

Warum Git?

Git ist das meistgenutzte Versionskontrollsystem und bietet:

  • Dezentrale Struktur
  • Schnelle Performance
  • Branching und Merging
  • Große Community

Wichtige Git-Befehle

Grundlegende Befehle

git init

Initialisiert ein neues Git-Repository

git clone

Klont ein bestehendes Repository

git add

Fügt Änderungen zur Staging-Area hinzu

git commit

Speichert Änderungen im Repository

Branch-Befehle

git branch

Zeigt und erstellt Branches

git checkout

Wechselt zwischen Branches

git merge

Führt Branches zusammen

git rebase

Basis eines Branches ändern

Remote-Befehle

git remote

Verwaltet Remote-Repositories

git fetch

Lädt Remote-Änderungen herunter

git pull

Lädt und merged Remote-Änderungen

git push

Lädt lokale Änderungen hoch

Best Practices

Commit-Messages

  • ✅ Prägnant und beschreibend
  • ✅ Im Imperativ schreiben
  • ✅ Änderungen klar beschreiben
  • ❌ Vage Beschreibungen vermeiden

Branching-Strategie

  • ✅ Feature Branches nutzen
  • ✅ Regelmäßig mergen
  • ✅ Konflikte zeitnah lösen
  • ❌ Direkt in main committen

Workflow

  • ✅ Regelmäßige kleine Commits
  • ✅ Pull vor Push
  • ✅ Code Review durchführen
  • ❌ Große, unübersichtliche Commits
