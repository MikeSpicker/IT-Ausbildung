Prozessverwaltung in Betriebssystemen

Was ist ein Prozess?

Ein Prozess ist ein Programm in Ausführung. Er besteht aus:

  • Programmcode

    Die auszuführenden Anweisungen

  • Daten

    Variablen und Arbeitsspeicher des Prozesses

  • Prozesszustandsinformation

    Programmzähler, Register, Stackpointer etc.

Prozesszustände

Zustand Beschreibung Übergang zu
Neu Prozess wird erstellt Bereit
Bereit Wartet auf CPU-Zuteilung Laufend
Laufend Wird aktuell ausgeführt Blockiert/Bereit/Beendet
Blockiert Wartet auf Ereignis/Ressource Bereit
Beendet Ausführung abgeschlossen -

Scheduling-Strategien

Präemptive Scheduling

  • Round Robin

    Jeder Prozess erhält ein Zeitquantum

  • Prioritätsbasiert

    Prozesse mit höherer Priorität werden bevorzugt

Nicht-präemptive Scheduling

  • First Come First Serve

    Prozesse werden in Ankunftsreihenfolge bearbeitet

  • Shortest Job First

    Kürzeste Prozesse werden zuerst ausgeführt

Übungsaufgaben

In welchen Zustand wechselt ein Prozess in folgenden Situationen?

  • Ein Prozess wartet auf eine Festplatten-Operation
  • Das Zeitquantum eines Prozesses ist abgelaufen
  • Ein Prozess wird neu gestartet
  • Festplatten-Operation: Blockiert (wartet auf I/O)
  • Zeitquantum abgelaufen: Bereit (wartet auf neue CPU-Zeit)
  • Neustart: Neu → Bereit
